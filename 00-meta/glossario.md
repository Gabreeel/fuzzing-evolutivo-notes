# Glossário

## Termos Fundamentais

### Fuzzing
Técnica de teste de software que consiste em fornecer entradas aleatórias ou semi-aleatórias a um programa para encontrar bugs, vulnerabilidades ou comportamentos inesperados.

### Coverage (Cobertura)
Métrica que indica quais partes do código foram executadas durante o teste. Tipos comuns:
- **Line Coverage**: cobertura de linhas
- **Branch Coverage**: cobertura de branches/decisões
- **Path Coverage**: cobertura de caminhos

### Algoritmo Genético (GA - Genetic Algorithm)
Técnica de otimização inspirada na evolução biológica, usando operadores como seleção, crossover e mutação para evoluir soluções.

### Programação Genética (GP - Genetic Programming)
Variante de algoritmo genético onde os indivíduos são programas ou expressões, não apenas vetores de parâmetros.

### Fitness Function
Função que avalia a qualidade de um indivíduo (input/teste) no contexto evolutivo.

### Mutation
Operador que introduz pequenas alterações aleatórias em um indivíduo.

### Crossover
Operador que combina partes de dois indivíduos para criar novos descendentes.

### Grammar-based Fuzzing
Fuzzing que usa gramáticas formais para gerar inputs válidos ou semi-válidos.

### Differential Fuzzing
Técnica que compara o comportamento de múltiplas implementações com as mesmas entradas para detectar inconsistências.

### Benchmark
Conjunto de programas e testes padronizados usados para avaliar ferramentas de fuzzing.

### CVE (Common Vulnerabilities and Exposures)
Sistema de identificação de vulnerabilidades de segurança conhecidas.
